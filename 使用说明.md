# 课程自动录制器 - 使用说明

## 功能特点
- ✓ 自动全屏录制课程视频
- ✓ 智能检测课程结束（通过播放器状态变化）
- ✓ 自动点击"下一节"按钮
- ✓ 自动保存并继续下一节录制
- ✓ 文件自动编号（从1开始）

## 安装步骤

### 1. 安装Python依赖
在命令行中运行：
```bash
pip install -r requirements.txt
```

或手动安装：
```bash
pip install opencv-python numpy pyautogui
```

### 2. 安装FFmpeg
**方法A - 使用包管理器（推荐）：**
- Windows Chocolatey: `choco install ffmpeg`
- 或从官网下载: https://ffmpeg.org/download.html

**方法B - 简单安装：**
1. 下载ffmpeg Windows构建版本
2. 解压到任意目录（如 `C:\ffmpeg`）
3. 将 `C:\ffmpeg\bin` 添加到系统PATH环境变量
4. 或直接将 `ffmpeg.exe` 复制到本脚本所在目录

### 3. 验证安装
在命令行运行：
```bash
ffmpeg -version
```
如果能显示版本信息，说明安装成功。

## 使用方法

### 首次运行（配置模式）
```bash
python auto_course_recorder.py
```

按提示完成以下配置：
1. **设置"下一节"按钮位置**
   - 将鼠标移动到"下一节"按钮上
   - 在命令行输入 `y` 并回车来记录位置

2. **设置"开始播放"按钮位置** ⭐ 新增
   - 将鼠标移动到"开始播放/播放"按钮上
   - 在命令行输入 `y` 并回车来记录位置

3. **设置播放器检测区域**
   - 将鼠标移动到检测区域左上角
   - 在命令行输入 `y` 并回车来记录左上角
   - 将鼠标移动到检测区域右下角
   - 在命令行输入 `y` 并回车来记录右下角
   - 这个区域用于检测课程是否结束

4. **学习"停止播放"状态** ⭐ 重要
   - **等待一节课程播放完成**
   - 确保播放器已经停止播放
   - 在命令行输入 `y` 并回车来捕获参考图像
   - 这个参考图像用于检测课程是否结束

### 日常使用
配置完成后，以后运行会自动加载配置：
```bash
python auto_course_recorder.py
```

按提示选择：
- 是否重新配置（一般选n）
- 起始文件编号（直接回车使用默认值1）

### 停止录制
按 `Ctrl + C` 停止录制

## 工作原理

**状态机设计：**
- `状态1 (playing_state=1)`：播放中
- `状态0 (playing_state=0)`：已停止（等待开始）

**工作流程：**

1. **初始状态 = 1（播放中）**

2. **检测到课程结束时：**
   - 如果 `状态=1`：
     * 停止录制
     * 点击"下一节"按钮
     * 状态变为 0
   - 如果 `状态=0`：
     * 点击"开始播放"按钮
     * 状态变为 1
     * 开始录制新文件

3. **录制阶段**
   - 使用ffmpeg全屏录制，30fps
   - 保存为MP4格式（H.264编码）

4. **检测阶段**
   - 每5秒截取检测区域的图像
   - 与"停止播放"参考图像对比相似度
   - 如果与停止图像相似（差异小），触发状态切换

**示例流程：**
```
[开始录制课程1] → 状态=1
  ↓
[检测到停止] → 点击"下一节" → 状态=0
  ↓
[检测到停止] → 点击"开始播放" → 状态=1 → [开始录制课程2]
  ↓
[检测到停止] → 点击"下一节" → 状态=0
  ↓
[检测到停止] → 点击"开始播放" → 状态=1 → [开始录制课程3]
  ↓
...循环...
```

## 配置文件
- 配置保存在 `recorder_config.json`
- 参考图像保存在 `stopped_reference.png`（停止播放时的参考图像）
- 如需重新配置，删除这些文件或运行时选择重新配置

## 录制文件
- 保存在 `recordings/` 目录
- 命名格式: `课程_1_20240316_143020.mp4`
- 格式: 课程_编号_时间戳.mp4

## 调整检测灵敏度

如果检测不准确，可以修改 `auto_course_recorder.py` 中的参数：

```python
# 第184行左右
def check_course_finished(self, threshold=2.0):  # 调整threshold值
```

**新逻辑：检测与停止图像的相似度**
- diff_score < threshold：判断为停止（课程结束）
- diff_score 范围：0（完全相同）到 255（完全不同）

**阈值调整建议：**
- **阈值 = 0**：完全一模一样才判断（最严格，可能因为像素级噪声无法识别）
- **阈值 = 1-3**：几乎一模一样（极严格，默认值2）
- **阈值 = 5-10**：非常严格，允许微小差异
- **阈值 = 15-30**：适中，允许一定差异
- **阈值 > 30**：宽松，容易误判

**根据实际情况调整：**
- 如果检测不到结束（太严格）：增大阈值（如改为5或10）
- 如果误判（还在播放就判断结束）：减小阈值（如改为1）

**默认值：2（极严格模式，几乎要一模一样）**

## 注意事项

1. **配置前准备** ⭐ 重要
   - 先等待一节课程播放完成
   - 确保播放器处于停止状态
   - 再进行步骤4的参考图像捕获
   - 这样可以准确捕获停止播放时的特征

2. **录制前准备**
   - 确保课程页面已打开
   - 检查"下一节"按钮位置正确
   - 检查"开始播放"按钮位置正确
   - 确保检测区域能反映播放器状态变化

3. **录制期间**
   - 不要遮挡检测区域
   - 不要移动浏览器窗口
   - 保持脚本窗口在后台

4. **磁盘空间**
   - 全屏录制约占用 1-2GB/分钟
   - 确保有足够磁盘空间

5. **性能**
   - 录制期间CPU占用较高
   - 建议关闭不必要的程序

6. **状态说明**
   - 脚本会在控制台显示当前状态（播放中/等待开始）
   - 每次状态切换都会打印日志
   - 如果状态卡住，检查按钮位置是否正确

## 常见问题

**Q: 脚本无法检测到课程结束？**
A: 增大阈值（如从30改为40），或确保参考图像是在播放器真正停止时捕获的。

**Q: 脚本在播放时就判断结束？**
A: 减小阈值（如从30改为20），或重新捕获参考图像确保是停止状态。

**Q: 点击"下一节"后没有自动开始录制？**
A: 检查"开始播放"按钮位置是否配置正确，状态会在点击下一节后变为0，等待检测到停止时会点击开始播放。

**Q: 状态一直停留在"等待开始"？**
A: 说明一直检测到停止状态，检查开始播放按钮位置，或手动点击一下让播放开始。

**Q: 录制文件很大？**
A: 可以修改脚本中的CRF值（第233行），增大到28可减小文件大小。

**Q: 如何重新捕获参考图像？**
A: 删除 `stopped_reference.png` 文件和 `recorder_config.json`，然后重新运行脚本进行配置。

**Q: 如何跳过某一节？**
A: 按 `Ctrl+C` 停止录制，手动点击下一节后重新运行，修改起始编号。
